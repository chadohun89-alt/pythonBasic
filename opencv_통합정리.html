① 파일 읽기

1이미지 읽기 → cv2.imread()

img = cv2.imread("opencv_study/images/coin.png")
인수타입설명

cv2.IMREAD_UNCHANGED → 알파 포함 4채널
반환값 → NumPy 배열 (shape: 세로 × 가로 × 채널)
OpenCV는 BGR 순서 사용 (일반 RGB와 반대)
② 가공

1크기변환 → cv2.resize()

인수설명

fx, fy 선택배율로 지정할 때 사용. 0.5 = 절반

2반전 → cv2.flip()

src원본 이미지 배열
flipCode 1 → 좌우 반전 (수직 축 기준)
0 → 상하 반전 (수평 축 기준)

3색상 처리

3-1흑백변환 → cv2.cvtColor()

gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
인수설명
src원본 이미지
code COLOR_BGR2GRAY → 컬러→흑백
COLOR_BGR2RGB → BGR→RGB
COLOR_BGR2HSV → BGR→HSV
흑백으로 바꾸면 shape가 (h, w)로 줄어든다 (채널 없어짐)

3-2경계1 → cv2.threshold()

_, thresh = cv2.threshold(blur, 131, 255, cv2.THRESH_BINARY_INV)
인수설명
src흑백(1채널) 이미지
thresh기준 밝기값 (0~255)
maxval기준 초과 픽셀에 넣을 값 (보통 255)
type THRESH_BINARY → 기준 초과 → maxval, 이하 → 0
THRESH_BINARY_INV → 반전. 배경이 밝고 대상이 어두울 때 사용
반환값 2개 → (실제 사용된 threshold 값, 결과 이미지). 첫 번째는 보통 _로 무시

3-3경계2 → cv2.Canny()

edge = cv2.Canny(blur, 50, 150)
인수설명
image흑백 이미지 (GaussianBlur 적용 후 권장)
threshold1낮은 기준값 → 이 이하는 edge 제외
threshold2높은 기준값 → 이 이상은 확실한 edge
기준값을 낮추면 edge 많이 검출, 높이면 뚜렷한 경계만 남는다

3-4윤곽선 → findContours / drawContours / contourArea

① cv2.findContours()
contours, hier = cv2.findContours(thresh, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
인수설명
image이진화 또는 Canny 처리된 이미지
mode RETR_EXTERNAL → 가장 바깥쪽 윤곽선만
RETR_LIST → 모든 contour
RETR_TREE → 모든 contour + 계층관계 저장
method CHAIN_APPROX_SIMPLE → 필요한 좌표만 저장 (효율적)
CHAIN_APPROX_NONE → 모든 픽셀 좌표 저장
② cv2.drawContours()
cv2.drawContours(result, contours, -1, (0, 0, 255), 2)
인수설명
image그릴 대상 이미지 (원본 copy 권장)

contourIdx그릴 contour 인덱스. -1이면 전체
colorBGR 색상 튜플
thickness선 두께(px). -1이면 내부 채우기
③ cv2.contourArea()
area = cv2.contourArea(cnt) if area < 2500: continue # 작은 것 제외
반환값: 해당 contour의 픽셀 면적 (float). 노이즈나 작은 객체 필터링에 활용

인수타입설명
filename str / int 영상 파일 경로 또는 카메라 인덱스 (0 = 웹캠)
주요 메서드

cap.release() → 영상 파일 닫기 (메모리 해제)
width = cap.get(cv2.CAP_PROP_FRAME_WIDTH) height = cap.get(cv2.CAP_PROP_FRAME_HEIGHT) fps = cap.get(cv2.CAP_PROP_FPS) total = cap.get(cv2.CAP_PROP_FRAME_COUNT)
cap.get() / cap.set() 주요 속성의미
CAP_PROP_POS_FRAMES현재 위치 (프레임 번호)
CAP_PROP_POS_MSEC현재 위치 (밀리초)
CAP_PROP_FPS초당 프레임 수
CAP_PROP_FRAME_WIDTH영상 너비
CAP_PROP_FRAME_HEIGHT영상 높이
CAP_PROP_FRAME_COUNT전체 프레임 수

2FPS 기반 딜레이 계산NEW

fps = cap.get(cv2.CAP_PROP_FPS) delay = int(1000 / fps) # waitKey에 넣을 딜레이(ms)
waitKey(30)은 고정값(약 33fps)이라 원본 FPS와 맞지 않을 수 있다
int(1000 / fps)로 계산하면 영상 고유 FPS에 맞게 재생된다
예: 30fps → 33ms / 60fps → 16ms / 24fps → 41ms
waitKey(delay * 2)처럼 배수를 주면 영상을 느리게 재생할 수 있다

4-2모션 감지NEW

① 이전 프레임 저장 패턴

pre_frame = None # 루프 밖에서 초기화 while True: ret, frame = cap.read() gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY) if pre_frame is None: # 첫 프레임은 저장만 pre_frame = gray continue diff = cv2.absdiff(pre_frame, gray) pre_frame = gray # 루프 끝에서 현재 → 이전으로 갱신
pre_frame은 루프 밖에서 None으로 초기화
루프 끝에서 현재 프레임으로 갱신해야 연속 비교가 된다
첫 프레임은 비교 대상이 없으므로 continue로 건너뛴다

② 프레임 차이 절대값 → cv2.absdiff()

diff = cv2.absdiff(pre_frame, gray)
인수타입설명
src1 ndarray 이전 프레임 (그레이스케일 권장)
src2 ndarray 현재 프레임 (같은 크기·타입)
반환값: 두 이미지의 픽셀 차이 절대값
움직인 부분 → 밝은 값, 변화 없는 배경 → 0(검정)
이후 threshold()로 이진화하면 움직임 마스크가 된다

③ 영역 팽창 → cv2.dilate()

thresh = cv2.dilate(thresh, None, iterations=2)
인수설명
src팽창할 이진 이미지
kernelNone이면 3×3 사각형 커널 자동 사용
iterations 선택팽창 반복 횟수. 클수록 영역이 더 넓어진다
흩어진 흰색 픽셀을 부풀려 인접 영역을 하나로 합쳐준다
찢어진 윤곽선을 연결해 contour 탐지가 더 잘 된다

4-3모폴로지 연산NEW

① 커널 생성 → np.ones()

import numpy as np kernel = np.ones((5, 5), np.uint8)
인수설명
shape커널 크기 (행, 열). 홀수 권장 (3×3, 5×5). 클수록 효과 강함
dtypenp.uint8 고정

② 노이즈 제거 → cv2.MORPH_OPEN

opened = cv2.morphologyEx(thresh, cv2.MORPH_OPEN, kernel, iterations=1)
OPEN = 침식(Erosion) → 팽창(Dilation)
작은 노이즈 점을 먼저 깎아내고 다시 복원 → 작은 노이즈 제거, 큰 영역은 유지

③ 끊어진 영역 연결 → cv2.MORPH_CLOSE

linked = cv2.morphologyEx(opened, cv2.MORPH_CLOSE, kernel, iterations=10)
CLOSE = 팽창(Dilation) → 침식(Erosion)
먼저 부풀려 인접 영역을 붙인 뒤 다시 깎아 복원 → 끊긴 윤곽 연결, 구멍 메우기
권장 순서: threshold → MORPH_OPEN → MORPH_CLOSE → dilate → findContours

④ cv2.morphologyEx() 공통 파라미터

인수타입설명
src ndarray 이진 이미지 (threshold 결과)
op int MORPH_OPEN → 노이즈 제거
MORPH_CLOSE → 끊긴 영역 연결
MORPH_ERODE → 침식만 / MORPH_DILATE → 팽창만
kernel ndarray np.ones()로 생성한 커널
iterations 선택 int 연산 반복 횟수. 클수록 효과 강함 (기본값 1)

4-4배경 제거기 (MOG2)NEW

① 생성 → cv2.createBackgroundSubtractorMOG2()

bgdelete = cv2.createBackgroundSubtractorMOG2( history=300, varThreshold=40, detectShadows=True )
인수설명
history 선택 배경 학습에 사용할 과거 프레임 수. 클수록 배경을 천천히 갱신 (기본 500)
varThreshold 선택 변화 판단 임계값. 클수록 덜 민감 → 작은 움직임 무시 (기본 16)
detectShadows 선택 True이면 그림자를 회색(127)으로 표시. False이면 흰색(255)으로
루프 밖에서 한 번만 생성하고, 루프 안에서 .apply()를 프레임마다 호출한다

② 마스크 추출 → bgdelete.apply()

fmask = bgdelete.apply(frame, learningRate=0.005) # 그림자(127)를 배경으로 처리 → 전경(255)만 남기기 _, thresh = cv2.threshold(fmask, 200, 255, cv2.THRESH_BINARY)
인수설명
image현재 프레임 (BGR 컬러 그대로 입력 가능)
learningRate 선택 배경 갱신 속도 (0.0~1.0)
-1 → 자동 / 0 → 배경 고정 / 0.005 → 천천히 갱신
반환값 fmask 픽셀 의미
전경(움직임) = 255 / 배경 = 0 / 그림자(detectShadows=True) = 127
threshold(fmask, 200, ...)로 그림자를 배경으로 처리해 전경만 남긴다
