▲이미지 처리 흐름

① 모션 분석 핵심 함수

1프레임 차이 분석

2-1이전 프레임 저장 패턴

pre_frame = None # 루프 밖에서 초기화 while True: ret, frame = cap.read() gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY) if pre_frame is None: # 첫 프레임은 저장만 pre_frame = gray continue diff = cv2.absdiff(pre_frame, gray) # ... (threshold, dilate, findContours) pre_frame = gray # 루프 끝에서 현재 → 이전으로 갱신
핵심 패턴
pre_frame은 루프 밖에서 None으로 초기화
루프 끝에서 현재 프레임으로 갱신해야 연속 비교가 된다
첫 프레임은 비교 대상이 없으므로 continue로 건너뛴다

2-2프레임 차이 절대값 → cv2.absdiff()

diff = cv2.absdiff(pre_frame, gray)
인수타입설명
src1 ndarray 이전 프레임 (그레이스케일 권장)
src2 ndarray 현재 프레임 (같은 크기·타입)
반환값: 두 이미지의 픽셀 차이 절대값
움직인 부분 → 밝은 값, 변화 없는 배경 → 0(검정)
이후 threshold()로 이진화하면 움직임 마스크가 된다

2-3영역 팽창 → cv2.dilate()

thresh = cv2.dilate(thresh, None, iterations=2)
인수설명
src팽창할 이진 이미지
kernel None이면 3×3 사각형 커널 자동 사용
iterations 선택 팽창 반복 횟수. 클수록 영역이 더 넓어진다
흩어진 흰색 픽셀을 부풀려 인접 영역을 하나로 합쳐준다
찢어진 윤곽선을 연결해 contour 탐지가 더 잘 된다
과하면 여러 객체가 하나로 합쳐질 수 있으니 iterations 값 조정 필요
② 텍스트 표시

2텍스트 그리기 → cv2.putText()

cv2.putText( result, "motion", (x, y - 10), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (0, 255, 0), 2 )
인수타입설명
img ndarray 텍스트를 그릴 이미지
text str 출력할 문자열 (한글 불가 → 영문 사용)
org tuple 텍스트 왼쪽 하단 기준점 (x, y). bounding box 위에 표시하려면 (x, y-10)
fontFace int FONT_HERSHEY_SIMPLEX 가장 많이 씀
fontScale float 글자 크기 배율. 0.5~1.0 정도가 적당
color tuple BGR 색상. (0, 255, 0) = 초록
thickness int 글자 선 두께(px)
org는 텍스트의 왼쪽 아래 기준점 → bounding box 위에 띄우려면 (x, y - 10)
OpenCV는 한글 폰트를 기본 지원하지 않아 영문·숫자만 출력 가능
③ 모폴로지 연산 (morphologyEx)

3모폴로지 연산

3-1커널(kernel) 생성 → np.ones()

import numpy as np kernel = np.ones((5, 5), np.uint8)
인수설명
shape커널 크기 (행, 열). 홀수 권장 (3×3, 5×5, 7×7). 클수록 효과 강함
dtypenp.uint8 고정. 0~255 정수형
커널이 클수록 연산 효과가 강해진다. 5×5는 3×3보다 더 강하게 노이즈 제거 또는 연결
cv2.dilate()에서 None을 쓰면 3×3 커널이 자동 적용됨

3-2노이즈 제거 → cv2.MORPH_OPEN

opened = cv2.morphologyEx( thresh, cv2.MORPH_OPEN, kernel, iterations=1 )
OPEN = 침식(Erosion) → 팽창(Dilation)
작은 노이즈 점을 먼저 깎아내고 다시 복원
효과: 크기가 작은 노이즈 제거, 큰 영역은 모양 유지

3-3끊어진 영역 연결 → cv2.MORPH_CLOSE

linked = cv2.morphologyEx( opened, cv2.MORPH_CLOSE, kernel, iterations=10 )
CLOSE = 팽창(Dilation) → 침식(Erosion)
먼저 부풀려 인접 영역을 붙인 뒤 다시 깎아 원래 크기 복원
효과: 영역 안의 구멍·틈새 메우기, 끊긴 윤곽 연결
iterations=10은 사람 몸통처럼 크게 끊어진 경우를 이을 때 사용

3-4cv2.morphologyEx() 공통 파라미터

인수타입설명
src ndarray 이진 이미지 (threshold 결과)
op int MORPH_OPEN → 노이즈 제거
MORPH_CLOSE → 끊긴 영역 연결
MORPH_ERODE → 침식만
MORPH_DILATE → 팽창만
kernel ndarray np.ones()로 생성한 커널
iterations 선택 int 연산 반복 횟수. 클수록 효과 강함 (기본값 1)
권장 순서:
threshold → MORPH_OPEN(노이즈 제거) → MORPH_CLOSE(영역 연결) → dilate(영역 확장) → findContours
④ 배경 제거기 (BackgroundSubtractorMOG2)

4MOG2 배경 제거기

4-1배경 제거기 생성 → cv2.createBackgroundSubtractorMOG2()

bgdelete = cv2.createBackgroundSubtractorMOG2( history=300, varThreshold=40, detectShadows=True )
인수설명
history 선택 배경 학습에 사용할 과거 프레임 수. 클수록 배경을 천천히 갱신 (기본 500)
varThreshold 선택 변화 판단 임계값. 클수록 덜 민감 → 작은 움직임 무시 (기본 16)
detectShadows 선택 True이면 그림자를 회색(127)으로 표시. False이면 그림자도 흰색(255)으로
루프 밖에서 한 번만 생성하고, 루프 안에서 .apply()를 프레임마다 호출한다
absdiff 방식과 달리 pre_frame을 직접 관리하지 않아도 된다

4-2마스크 추출 → bgdelete.apply()

fmask = bgdelete.apply(frame, learningRate=0.005) # 이후: 이진화로 그림자(127) 제거 → 전경(255)만 남기기 _, thresh = cv2.threshold(fmask, 200, 255, cv2.THRESH_BINARY)
인수설명
image 현재 프레임 (BGR 컬러 그대로 입력 가능)
learningRate 선택 배경 갱신 속도 (0.0 ~ 1.0)
-1 → 자동, 0 → 배경 고정, 0.005 → 천천히 갱신
카메라 고정 영상: 0.001~0.01 권장
반환값 fmask 픽셀 의미
전경(움직이는 부분) = 255(흰색)
배경 = 0(검정)
그림자(detectShadows=True일 때) = 127(회색)

threshold(fmask, 200, 255, ...)로 그림자(127)를 배경으로 처리해 전경만 남긴다
⑤ 모션 감지 방법 비교

5absdiff 방식 vs MOG2 배경 제거기

항목 absdiff 방식 MOG2 배경 제거기
원리 연속 두 프레임의 픽셀 차이 과거 N프레임으로 배경 모델링 후 전경 분리
구현 방식 pre_frame 직접 관리 bgdelete.apply(frame) 한 줄
카메라 흔들림 취약 (조금만 흔들려도 전체 감지) 상대적으로 강건
조명 변화 대응 취약 history로 배경 적응 가능
그림자 처리 없음 (그림자도 움직임으로 감지) detectShadows=True로 분리 가능
적합한 상황 고정 카메라, 조명 안정, 간단 구현 조명 변화 있는 환경, 정밀 감지 필요 시
두 방법 모두 이후 처리는 동일:
threshold → 모폴로지(OPEN/CLOSE) → dilate → findContours → boundingRect → putText
